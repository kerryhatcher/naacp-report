# Report Data Pipeline + React SPA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a `uv`-managed Python data pipeline that compiles report content into static JSON, and a React SPA that renders it, proven end-to-end with the county boards deliverable.

**Architecture:** `generator/` (uv project) reads Markdown+frontmatter content and writes JSON directly into `web/public/data/`. `web/` (Vite + React + TypeScript) is a client-only SPA that fetches that JSON via TanStack Query as if it were a REST API — there is no live backend, ever. The two projects only share the JSON file contract.

**Tech Stack:** Python 3.13, uv, python-frontmatter, markdown, pytest, ruff · Vite, React, TypeScript, React Router, TanStack Query, Tailwind CSS v4 · GitHub Actions + GitHub Pages.

## Global Constraints

- Python: `requires-python = ">=3.13"`.
- Use `uv` for every Python command (`uv run`, `uv add`, `uv add --dev`) — never bare `python`/`pip`.
- Run `uv run ruff check .` and confirm a clean pass before every Python commit.
- Frontend stack is exactly: Vite + React + TypeScript + React Router + TanStack Query + Tailwind CSS — no additional UI/component library.
- Every builder's JSON output follows the list + detail shape: list files stay lean (index/table fields only), detail files carry the full record.
- `web/public/data/` is a build artifact — git-ignored, regenerated by `uv run build.py`, never hand-edited or committed directly.
- v1 scope: only the `county_boards` builder is fully implemented; `turnout` and `demographics` are stubs that write `[]`.
- No test framework is configured for the React app in v1 — verify UI changes visually with the Playwright MCP tool and save screenshots to the git-ignored `screenshots/` folder.
- Deployment target is GitHub Pages via GitHub Actions, publishing `web/dist/`.

- [ ] **Step 4: Implement the builder**

`generator/builders/county_boards.py`
```python
import json
from pathlib import Path

import frontmatter
import markdown

CONTENT_DIR = Path(__file__).parent.parent / "content" / "county-boards"
OUTPUT_DIR = Path(__file__).parent.parent.parent / "web" / "public" / "data"


def parse_county(md_path: Path) -> dict:
    post = frontmatter.load(md_path)
    return {
        "slug": md_path.stem,
        "name": post.get("county", md_path.stem),
        "members": post.get("members"),
        "selection_method": post.get("selection_method"),
        "meeting_schedule": post.get("meeting_schedule"),
        "body_html": markdown.markdown(post.content),
    }


def build(content_dir: Path = CONTENT_DIR, output_dir: Path = OUTPUT_DIR) -> None:
    detail_dir = output_dir / "counties"
    detail_dir.mkdir(parents=True, exist_ok=True)

    summaries = []
    for md_path in sorted(content_dir.glob("*.md")):
        county = parse_county(md_path)
        summaries.append(
            {
                "slug": county["slug"],
                "name": county["name"],
                "members": county["members"],
                "selection_method": county["selection_method"],
            }
        )
        (detail_dir / f"{county['slug']}.json").write_text(json.dumps(county, indent=2))

    (output_dir / "counties.json").write_text(json.dumps(summaries, indent=2))
```

export function CountiesList() {

- [ ] **Step 2: Write `CountyDetail`**

`web/src/routes/CountyDetail.tsx`
```tsx
import { useQuery } from '@tanstack/react-query'
import { useParams } from 'react-router-dom'
import { fetchCounty } from '../lib/api'

export function CountyDetail() {
  const { slug } = useParams<{ slug: string }>()
  const { data, isLoading, error } = useQuery({
    queryKey: ['county', slug],
    queryFn: () => fetchCounty(slug!),
    enabled: Boolean(slug),
  })

  if (isLoading) return <p className="p-4">Loading…</p>
  if (error) return <p className="p-4 text-red-600">Failed to load this county.</p>

  return (
    <div className="p-4">
      <h1 className="text-2xl font-bold">{data!.name} County Board of Elections</h1>
      <dl className="mt-4 grid grid-cols-[max-content_1fr] gap-x-4 gap-y-1">
        <dt className="font-semibold">Members</dt>
        <dd>{data!.members ?? '—'}</dd>
        <dt className="font-semibold">Selection method</dt>
        <dd>{data!.selection_method ?? '—'}</dd>
        <dt className="font-semibold">Meeting schedule</dt>
        <dd>{data!.meeting_schedule ?? '—'}</dd>
      </dl>
      <div className="mt-4 prose" dangerouslySetInnerHTML={{ __html: data!.body_html }} />
    </div>
  )
}
```

### Task 9: GitHub Actions build + deploy to GitHub Pages

**Files:**
- Create: `.github/workflows/deploy.yml`
- Test: manual (YAML syntax validation locally; full validation happens on first push)

- [ ] **Step 1: Write the workflow**

`.github/workflows/deploy.yml`
```yaml
name: Deploy report site

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v3

      - name: Build report data
        working-directory: generator
        run: uv run build.py

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install web dependencies
        working-directory: web
        run: npm ci

      - name: Build web app
        working-directory: web
        run: npm run build

      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: web/dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

- [ ] **Step 2: Validate YAML syntax locally**

Run (from repo root): `uv run --with pyyaml python -c "import yaml; yaml.safe_load(open('.github/workflows/deploy.yml')); print('valid')"`
Expected: `valid`

```bash
git add .github/workflows/deploy.yml
git commit -m "ci: deploy generated report to GitHub Pages on push to main"
```
